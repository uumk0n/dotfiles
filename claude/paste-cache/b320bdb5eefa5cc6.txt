⏺ Что делать на фронте

  Google Sign-In

  1. Установить SDK:
  # React Native
  pnpm add @react-native-google-signin/google-signin
  2. Настроить:
  GoogleSignin.configure({
    iosClientId: 'YOUR_IOS_CLIENT_ID',   // отдельный Client ID для iOS
    webClientId: 'YOUR_WEB_CLIENT_ID',   // тот же что в GOOGLE_CLIENT_ID на бэке
  });
  ▎ Важно: для iOS нужен отдельный iOS Client ID из Google Console. webClientId — это тот что ты уже добавил на бэк.
  3. Кнопка логина:
  const { idToken } = await GoogleSignin.signIn();

  const res = await api.post('/auth/oauth/google', { idToken });
  // res.data = { status: 'SUCCESS', payload: { accessToken, refreshToken } }

  ---
  Apple Sign-In (iOS only)

  1. Установить:
  pnpm add @invertase/react-native-apple-authentication
  2. Кнопка логина:
  const credential = await appleAuth.performRequest({
    requestedOperation: appleAuth.Operation.LOGIN,
    requestedScopes: [appleAuth.Scope.EMAIL, appleAuth.Scope.FULL_NAME],
  });

  await api.post('/auth/oauth/apple', {
    identityToken: credential.identityToken,
    firstName: credential.fullName?.givenName,   // только при первом входе
    lastName: credential.fullName?.familyName,
  });
  ▎ fullName приходит только при первом входе — потом будет null. Бэк это учитывает.

  ---
  Хранение токенов

  После успешного ответа сохрани accessToken и refreshToken в Keychain (не в AsyncStorage):
  pnpm add react-native-keychain

  ---
  Что важно

  - Apple Sign-In работает только на реальном устройстве, не в симуляторе
  - Google требует добавить google-services.json (Android) и GoogleService-Info.plist (iOS) в проект
  - accessToken от нашего бэка — обычный JWT, живёт 1 час; refreshToken — 30 дней. Когда accessToken протухнет — используй POST /auth/refresh-token (уже существующий эндпоинт)
