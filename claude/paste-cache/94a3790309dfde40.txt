  Add support for multiple crypto networks in /crypto-addresses

  Currently the crypto addresses API only supports TRON. We need to extend it to support the following networks:

  - TRON
  - ETHEREUM
  - BASE
  - SOLANA
  - POLYGON

  ---
  1. Extend CryptoNetwork enum

  Add the new values to the existing enum:
  TRON, ETHEREUM, BASE, SOLANA, POLYGON

  ---
  2. Update address validation per network

  The endpoint POST /crypto-addresses currently validates the address as a TRON address. It should validate based on the network field
   in the request body:

  ┌──────────┬─────────────────────────────────────────────────────────────────────┐
  │ Network  │                           Validation rule                           │
  ├──────────┼─────────────────────────────────────────────────────────────────────┤
  │ TRON     │ Starts with T, exactly 34 chars, base58, valid Base58Check checksum │
  ├──────────┼─────────────────────────────────────────────────────────────────────┤
  │ ETHEREUM │ Starts with 0x, exactly 42 chars, valid hex                         │
  ├──────────┼─────────────────────────────────────────────────────────────────────┤
  │ BASE     │ Same as Ethereum                                                    │
  ├──────────┼─────────────────────────────────────────────────────────────────────┤
  │ POLYGON  │ Same as Ethereum                                                    │
  ├──────────┼─────────────────────────────────────────────────────────────────────┤
  │ SOLANA   │ Base58-encoded, decodes to exactly 32 bytes                         │
  └──────────┴─────────────────────────────────────────────────────────────────────┘

  ---
  3. Update error codes

  Currently on invalid address the API returns:
  { "error": "INVALID_TRON_ADDRESS" }

  Please change to a generic error code:
  { "error": "INVALID_ADDRESS" }

  ---
  4. Request/response contract — no changes needed

  CreateCryptoAddressRequest stays the same, just network now accepts the expanded enum:
  {
    "network": "ETHEREUM",
    "address": "0x...",
    "label": "My wallet",
    "setAsDefault": false
  }
