⏺ Всё ясно. Вот обновлённый промпт с разделом про аватарку:

  ---
  Промпт для фронтенда: Авторизация через Apple

  Контекст

  Бэкенд уже полностью поддерживает Sign In with Apple. Нужно подключить его на стороне фронтенда по аналогии с уже работающим Google OAuth.

  ---
  API-эндпоинт

  POST /auth/oauth/apple

  Request body:
  {
    "identityToken": "eyJ...",   // обязательное — JWT от Apple SDK
    "firstName": "John",         // опциональное — Apple отдаёт ТОЛЬКО при первом входе
    "lastName":  "Doe"           // опциональное — Apple отдаёт ТОЛЬКО при первом входе
  }

  Success response (200):
  {
    "status": "SUCCESS",
    "payload": {
      "accessToken":  "...",
      "refreshToken": "...",
      "idToken":      "..."
    }
  }

  Error (401) — невалидный или просроченный identityToken.

  ---
  Важная особенность Apple OAuth

  Apple возвращает firstName и lastName только один раз — при самом первом входе пользователя. При повторных входах эти поля будут null/undefined. Нужно обязательно передать их в запрос, если они пришли от Apple SDK — иначе
  имя пользователя навсегда останется пустым.

  ---
  Аватарка пользователя

  Apple не предоставляет URL аватарки в identity token — это принципиальное ограничение платформы.

  Для сравнения: при Google OAuth бэкенд сам скачивает фото профиля из Google и сохраняет его в S3. При Apple OAuth этого не происходит — аватарка не устанавливается автоматически.

  Пользователь сможет добавить фото только вручную через профиль после входа. Фронтенду нужно корректно отображать состояние "без аватарки" (инициалы / placeholder) для пользователей, зарегистрировавшихся через Apple.

  ---
  Что нужно сделать

  1. Подключить Sign In with Apple SDK (зависит от платформы):
    - Web: @apple/appleid-auth или нативный AppleID.auth
    - React Native: @invertase/react-native-apple-authentication
    - iOS native: ASAuthorizationAppleIDProvider
  2. После получения credential от Apple SDK — достать:
    - identityToken (строка JWT)
    - givenName / familyName из fullName (только при первом входе, может быть null)
  3. Вызвать бэкенд-эндпоинт, передав все три поля (если firstName/lastName есть):
  const response = await api.post('/auth/oauth/apple', {
    identityToken: credential.identityToken,
    firstName: credential.fullName?.givenName ?? undefined,
    lastName:  credential.fullName?.familyName ?? undefined,
  });
  4. Сохранить токены из response.payload — так же, как после Google OAuth (accessToken, refreshToken, idToken).
  5. Обработать ошибки: 401 → показать сообщение «Не удалось войти через Apple», не делать повторных попыток.

  ---
  Аналогия с Google

  Реализация идентична Google OAuth, который уже работает. Разница только в:

  ┌──────────────┬────────────────────────────────┬───────────────────────────────────┐
  │              │             Google             │               Apple               │
  ├──────────────┼────────────────────────────────┼───────────────────────────────────┤
  │ SDK метод    │ signIn() → idToken             │ performRequests() → identityToken │
  ├──────────────┼────────────────────────────────┼───────────────────────────────────┤
  │ Поле запроса │ idToken                        │ identityToken                     │
  ├──────────────┼────────────────────────────────┼───────────────────────────────────┤
  │ Имя/фамилия  │ Приходят всегда из SDK         │ Только при первом входе           │
  ├──────────────┼────────────────────────────────┼───────────────────────────────────┤
  │ Аватарка     │ Бэкенд загружает автоматически │ Не загружается, нужен placeholder │
  └──────────────┴────────────────────────────────┴───────────────────────────────────┘
